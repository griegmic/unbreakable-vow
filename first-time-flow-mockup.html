First-Time User Flow — Assessment

Expert panel evaluation of whether the "I vow to..." pattern should apply to first-time users. Conclusion: the first-time flow is already strong. Only minor consistency tweaks recommended.

Panel Verdict: Don't redesign — refine

The first-time flow has something the old returning-user flow lacked: a bold headline that fills the viewport with purpose, example chips that solve the blank-page problem, and a CTA with ritual language. Applying the "I vow to..." pattern here would remove the product education and examples that first-timers need.

Two consistency tweaks only: (1) Rename "I WILL..." to "I vow to..." to match the product's ceremonial language. (2) Add the gold glow shadow to the CTA button for visual consistency with the new returning-user flow.

— WHAT CHANGES AND WHY —

Only two changes

  • "I WILL..." → "I vow to..." — Matches the returning-user flow language. "I will" is generic productivity-app copy. "I vow to" is ceremonial and distinct to this product. The label becomes italic, slightly larger, gold-tinted — consistent with the returning-user prompt.
  • CTA glow shadow — The gold gradient button gets the same `box-shadow: 0 12px 32px rgba(212,162,79,0.25)` as the returning-user "Set the terms →" button. Subtle visual lift that makes it feel more premium.

What stays the same (and why)

  • "Make a vow. Mean it." — This headline IS the product pitch. First-timers need it. Removing it for "I vow to..." would lose the hook.
  • Example chips — These solve the blank-page problem for first-timers. Returning users don't need them (they know what to type). First-timers do.
  • "Put money on it. Break it, it goes to charity." — One-line product education. Returning users know this. First-timers don't.
  • "I vow it" CTA text — This is stronger than "Next" or "Continue" for first-timers. It's the first commitment moment.
  • Card wrapper around input — First-timers need the visual boundary. The free-flowing textarea works for returning users who know the context.
  • All routing, auth, and data logic — Zero changes. The flow still goes to /refine or /stake based on vow quality.
