This is the recommended app-native vow maker journey: mobile-web copy and hierarchy as the baseline, with native-only upgrades where they are clearly better: contact picking, haptics, Apple Pay-first payment, and push permission timing.

Flow spine: Cinematic -> vow -> optional sharpen -> stake/date/cause -> witness -> phone OTP -> name if missing -> Apple Pay/card -> sealed/share -> dashboard -> active detail.
Rule: no payment UI on phone auth; no contact permission until the user taps Add judge; no auto-advancing celebration.

Product Notes

  • “Or start here” pills: six total, wrapping on the web-style mock but implemented as a horizontally scrollable/safe native row if vertical space is tight: Gym 3x this week, No alcohol 2 weeks, Delete TikTok, No phone til noon, Call my mom, 10k steps daily.
  • Smart date: infer from vow text. “this week” -> Sunday night, “tomorrow” -> tomorrow 9pm, “2 weeks” -> 14 days, “30 days” -> 30 days. If no date is inferred, show the compact date sheet before terms.
  • Sharpen: not part of the normal path. It appears only when the vow logic marks the vow as vague. It should not punish good-enough vows.
  • Witness: the strongest native improvement is the contact picker, but it must be user-initiated. This screen should feel like accountability, not permissions.
  • Auth/payment: phone OTP and payment are separate because they solve different anxieties. Twilio verifies identity; Apple Pay/card saves the stake method.
  • Full vow card: every post-seal state should show the vow, stake, judge, destination, and verdict date so the maker always knows exactly what is at stake.
