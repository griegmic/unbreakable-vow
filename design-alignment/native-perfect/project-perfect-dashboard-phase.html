Dashboard Recommendation

Main dashboard: Keep it action-led, with role filters only when there are multiple roles. If the user has one vow, show the vow. If they have vows plus judging/dares, show All, My vows, Judging, Dares.

Witnessing and dares: Do not make them feel like separate apps. Urgent witness obligations promote into “Needs you now”; the Judging/Dares filters reveal the fuller list. This is cleaner than sending people into too many side sections.

Native delta: Role filters are a native-leaning improvement over mobile web because app users return more often and need quick switching without losing the dashboard as home.

Menu system: The hamburger should be a safety net. It gets Make a vow, My vows, People I’m judging, Dares I sent, Dare someone, Settings, Help. No separate Quick Vow vs Guided entries; guided lives as a secondary option inside Make a vow.

Section choice: “People I’m judging” is clearer in the menu than “Witnessing” because the user’s job there is verdict/action. In the maker flow, keep “witness” because it feels more human and less legalistic.

Donald pass: The main screen avoids a tab trap, keeps the primary CTA visible, and uses urgent color only for real action. Remaining risk: if a user is primarily a witness and has no vows, the Witnessing module should be promoted above “Your vows.”
