Donald + Stripe Pass

Witness timing: Mobile web already uses time pressure in the witness list, but the immediate accepted screen is a calm confirmation. Native should preserve that emotional shape: accept first, countdown later when the witness returns.

Countdown placement: The timer belongs in witness dashboard/detail states because that is when the witness is managing responsibility. It should become more urgent only near verdict time.

Payment truth: Stripe SetupIntent saves a payment method for later. The app should not call this a “hold” unless the backend uses a PaymentIntent manual-capture hold. The strongest accurate copy is “No charge now. Only if you break it.”

Quick Vow flow: Pressing Stake should go to Add Payment only if the user does not already have a saved method. If they do, use a lightweight one-tap confirmation and then go straight to the existing Vow Created / Text Witness moment.
