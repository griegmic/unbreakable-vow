Batch 2 Scorecard

Delta from mobile web: web uses share/copy as the primary witness mechanic. Native uses Add a judge with contacts because this is a clear Native Advantage. Copy stays very close; the extra contact-permission copy is allowed native-only trust copy.

Delta from current native: current native jumps quickly into a generic contact picker. This version makes the social decision visible first, then shows contacts only after intent. It keeps share link and solo as quieter escape hatches.

Why this is best for native: choosing a witness should feel like accountability, not a permissions task. The empty state makes the job obvious; the sheet makes contact sync contextual and calm.

Mobile web recommendation: no change yet. Mobile web should remain share-first unless we later build a web contacts/address-book path. Native earns the deviation because contacts are materially better in-app.
